**Sprint Duration:** 10 working days (2 weeks)

## Day 1 — Typed Configuration Foundation (dry-rb)

### Task 1.1: Install dry-configurable & dry-struct
- **Gems:**
  ```ruby
  gem "dry-configurable", "~> 1.2"
  gem "dry-struct", "~> 1.6"
  gem "dry-types", "~> 1.7"
  gem "dry-validation", "~> 1.10"
  ```
- **Command:** `bundle install`
- **Deliverable:** dry-rb stack installed and loadable
- **Acceptance Criteria:** `bundle exec ruby -e "require 'dry-configurable'; puts Dry::Configurable::VERSION"` succeeds

### Task 1.2: Create `AppConfig` Root Class
- **File:** `app/config/app_config.rb`
- **Requirements:**
  - Uses `Dry::Configurable` with `config` block
  - Sections: `trading`, `broker`, `redis`, `database`, `ai`, `risk`, `features`
  - Each section has typed defaults and coercion
  - Example structure:
    ```ruby
    class AppConfig
      extend Dry::Configurable

      setting :trading, default: {} do
        setting :market_open, default: "09:15"
        setting :market_close, default: "15:30"
        setting :paper_mode, default: true, constructor: ->(v) { !!v }
      end

      setting :broker, default: {} do
        setting :client_id, default: ""
        setting :access_token, default: ""
        setting :base_url, default: "https://api.dhan.co"
      end
      # ... etc
    end
    ```
- **Deliverable:** Centralized configuration class
- **Acceptance Criteria:** `AppConfig.config.trading.market_open` returns `"09:15"`

### Task 1.3: Create `AppConfigValidator`
- **File:** `app/config/app_config_validator.rb`
- **Requirements:**
  - Uses `Dry::Validation::Contract`
  - Validates all required keys are present and non-empty
  - Validates numeric ranges (e.g., `max_open_positions > 0`)
  - Validates URL formats for broker and AI endpoints
  - Validates time formats (`HH:MM`)
- **Deliverable:** Validation contract that fails fast
- **Acceptance Criteria:** `AppConfigValidator.new.call(config).errors.to_h` returns empty hash for valid config; returns specific errors for invalid config

## Day A — TOTP Infrastructure & User Model

### Task A.1: Install Authentication Gems
- **Gems to add to `Gemfile`:**

  ```ruby
  gem "rqrcode", "~> 2.2"         # QR code generation for enrollment

  gem "jwt", "~> 2.9"             # Session tokens
  ```
- **Command:** `bundle install`
- **Deliverable:** All auth gems installed
- **Acceptance Criteria:** `bundle exec ruby -e "require 'rotp'; require 'rqrcode'; require 'bcrypt'; require 'jwt'"` succeeds

### Task A.2: Create `User` Model & Migration
- **Migration:** `db/migrate/xxx_create_users.rb`
  ```ruby
  create_table :users do |t|
    t.string :email, null: false, index: { unique: true }
    t.string :pin_digest, null: false
    t.string :totp_secret, null: false
    t.boolean :totp_enabled, default: false, null: false
    t.boolean :active, default: true, null: false
    t.datetime :last_login_at
    t.timestamps
  end
  ```
- **Model:** `app/models/user.rb`
  ```ruby
  class User < ApplicationRecord
    has_secure_password :pin, validations: false  # Custom PIN handling

    validates :email, presence: true, format: { with: URI::MailTo::EMAIL_REGEXP }
    validates :pin_digest, presence: true
    validates :totp_secret, presence: true

    before_validation :generate_totp_secret, on: :create

    def verify_pin(plain_pin)
      BCrypt::Password.new(pin_digest).is_password?(plain_pin)
    end

    def verify_totp(token)
      ROTP::TOTP.new(totp_secret).verify(token, drift_behind: 30, drift_ahead: 30)
    end

    def provisioning_uri(issuer: "AlgoScalper")
      ROTP::TOTP.new(totp_secret).provisioning_uri(email, issuer_name: issuer)
    end

    def qr_code_svg(issuer: "AlgoScalper")
      RQRCode::QRCode.new(provisioning_uri(issuer: issuer)).as_svg
    end

    private

    def generate_totp_secret
      self.totp_secret ||= ROTP::Base32.random
    end
  end
  ```
- **Deliverable:** User model with PIN + TOTP
- **Acceptance Criteria:** `User.create!(email: "test@example.com", pin: "123456")` succeeds; `user.verify_pin("123456")` returns `true`; `user.verify_totp(ROTP::TOTP.new(user.totp_secret).now)` returns `true`

  ```ruby
    belongs_to :user

    before_validation :generate_token, on: :create
    before_validation :set_expiry, on: :create

    scope :active, -> { where(revoked_at: nil).where("expires_at > ?", Time.current) }

    def revoke!
      update!(revoked_at: Time.current)
    end

    def revoked?
      revoked_at.present? || expires_at <= Time.current
    end

    private

    def generate_token
      self.token_digest ||= Digest::SHA256.hexdigest(SecureRandom.hex(32))
    end

    def set_expiry
      self.expires_at ||= 24.hours.from_now
    end
  end
  ```
- **Deliverable:** Token-based session management
- **Acceptance Criteria:** `Session.create!(user: user)` generates token; `session.revoked?` returns `false` initially

## Day B — Authentication Service & Controllers

### Task B.1: Create `AuthenticationService`
- **File:** `app/services/authentication_service.rb`
- **Requirements:**
  ```ruby
  class AuthenticationService < BaseService
    PIN_LENGTH = 6
    MAX_ATTEMPTS = 5
    LOCKOUT_DURATION = 30.minutes

    def initialize(email:, pin:, totp_token: nil)
      @email = email
      @pin = pin
      @totp_token = totp_token
    end

    def call
      user = User.find_by(email: @email, active: true)
      return failure(:user_not_found) unless user

      return failure(:account_locked) if locked?(user)

      unless user.verify_pin(@pin)
        record_failed_attempt(user)
        return failure(:invalid_pin)
      end

      if user.totp_enabled
        return failure(:totp_required) if @totp_token.blank?
        return failure(:invalid_totp) unless user.verify_totp(@totp_token)
      end

      clear_attempts(user)
      session = create_session(user)

      success(session: session, token: session.token_digest)
    end

    private

    def locked?(user)
      # Implement Redis-based attempt tracking
      attempts = RedisPool.general.with { |r| r.get("auth:attempts:#{user.id}")&.to_i || 0 }
      attempts >= MAX_ATTEMPTS
    end

    def record_failed_attempt(user)
      RedisPool.general.with do |r|
        key = "auth:attempts:#{user.id}"
        r.incr(key)
        r.expire(key, LOCKOUT_DURATION.to_i)
      end
    end

    def clear_attempts(user)
      RedisPool.general.with { |r| r.del("auth:attempts:#{user.id}") }
    end

    def create_session(user)
      Session.create!(
        user: user,
        ip_address: Current.ip_address,
        user_agent: Current.user_agent
      )
    end
  end
  ```
- **Deliverable:** Complete auth flow with brute-force protection
- **Acceptance Criteria:** 5 failed PIN attempts lock account for 30 minutes; correct PIN + TOTP creates session

### Task B.4: Create `Authenticate` Middleware
- **File:** `app/controllers/concerns/authenticate.rb`
  ```ruby
  module Authenticate
    extend ActiveSupport::Concern

    included do
      before_action :authenticate!
      before_action :set_current_attributes
    end

    private

    def authenticate!
      token = extract_token
      return render_unauthorized("Missing token") unless token

      session = Session.active.find_by(token_digest: Digest::SHA256.hexdigest(token))
      return render_unauthorized("Invalid or expired token") unless session

      Current.session = session
      Current.user = session.user
    end

    def extract_token
      auth_header = request.headers["Authorization"]
      return nil unless auth_header&.start_with?("Bearer ")

      auth_header.split(" ").last
    end

    def render_unauthorized(message)
      render json: { error: message }, status: :unauthorized
    end

    def set_current_attributes
      Current.ip_address = request.remote_ip
      Current.user_agent = request.user_agent
      Current.request_id = request.uuid
    end
  end
  ```

### Task C.1: Create `PinPolicy` Validator
- **File:** `app/validators/pin_policy.rb`
- **Requirements:**
  ```ruby
  class PinPolicy
    MIN_LENGTH = 6
    MAX_LENGTH = 6
    FORBIDDEN_PINS = %w[000000 111111 123456 654321 121212].freeze

    def self.valid?(pin)
      return false unless pin.is_a?(String)
      return false unless pin.length == MIN_LENGTH
      return false unless pin.match?(/\A\d+\z/)
      return false if FORBIDDEN_PINS.include?(pin)
      return false if sequential?(pin)
      return false if repeated?(pin)
      true
    end

    def self.errors(pin)
      return ["PIN must be #{MIN_LENGTH} digits"] unless pin.to_s.length == MIN_LENGTH
      return ["PIN must contain only digits"] unless pin.match?(/\A\d+\z/)
      return ["PIN is too common"] if FORBIDDEN_PINS.include?(pin)
      return ["PIN cannot be sequential"] if sequential?(pin)
      return ["PIN cannot be repeated digits"] if repeated?(pin)
      []
    end

    def self.sequential?(pin)
      digits = pin.chars.map(&:to_i)
      digits.each_cons(2).all? { |a, b| b == a + 1 } ||
      digits.each_cons(2).all? { |a, b| b == a - 1 }
    end

    def self.repeated?(pin)
      pin.chars.uniq.length == 1
    end
  end
  ```
- **Deliverable:** Strong PIN validation
- **Acceptance Criteria:** `PinPolicy.valid?("123456")` returns `false` (sequential); `PinPolicy.valid?("482915")` returns `true`
